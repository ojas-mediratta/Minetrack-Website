Introduction & Background

Estimating motion from visual input is a core problem in robotics and computer vision. Visual Odometry (VO) aims to recover an agent's trajectory by estimating relative transformations between consecutive image frames and composing them over time. Robust VO remains essential for navigation, state estimation, and autonomous control.

Classical VO pipelines rely on strategies like feature detection and geometric optimization. Foundational surveys categorize VO methods into feature-based and appearance-based approaches, highlighting persistent challenges such as drift accumulation and sensitivity to visual ambiguity [1]. More recent reviews emphasize the transition toward learning-based methods that replace hand-engineered features with neural representations, particularly in monocular settings where scale and depth ambiguities are significant challenges [2], [3]. These works motivate multi-stage architectures that separate representation learning from motion regression.

In this project, we propose MineTrack, a learning-based visual odometry framework implemented in Minecraft. Minecraft provides a controllable 3D environment with exact ground-truth pose information, enabling reproducible experimentation without sensor calibration or noise confounds. Using a custom mod for the game, we log per-frame position and orientation, allowing precise computation of relative transformation matrices. Our dataset will consist of recorded gameplay trajectories across diverse environments (e.g., natural terrain and built structures), captured at a fixed frame rate and resolution. MineTrack adopts a modular learning strategy that separates representation learning from motion estimation. We will experiment with several unsupervised preprocessing approaches such as autoencoder-based embedding models and other latent representation techniques to learn compact visual encodings of gameplay frames. These learned representations will then serve as input to a supervised neural network that predicts relative motion parameters, with outputs as transformation matrices, between consecutive frames.

By decoupling representation learning from pose regression, MineTrack investigates whether unsupervised visual embeddings encode sufficient geometric structure to support accurate trajectory reconstruction. Predicted transformations are sequentially composed and evaluated against ground-truth motion to quantify drift and trajectory error.

Problem Definition

Reliable motion estimation from visual input is a core requirement for autonomous navigation.
Visual Odometry estimates a systems trajectory by predicting the relative pose change between consecutive camera frames and composing those predictions over time.
especially under changes in environment appearance such as textures, lighting and motion patterns.
MineTrack tackles learning-based visual odometry in a controlled 3D environment.
Given two consecutive RGB frames from Minecraft, we aim to predict the relative motion of the agent between frames (translation and rotation).
By composing predicted frame-to-frame transforms over time, we reconstruct the full trajectory and measure drift versus ground truth.

Research Question

Do self-supervised visual embeddings learn geometry features that improve downstream relative pose regression and reduce trajectory drift?

Objectives

  • Learn compact visual embeddings from played minecraft frames
  • Predict frame to frame motion and translate to trajectories
  • Evaluate trajectory accuracy and drift vs ground truth across varied environments

Scope

Monocular RGB visual odometry in Minecraft with frame-to-frame pose regression and trajectory reconstruction via transform composition.

Motivation

Motivation: While conducting SLAM in Minecraft is an impressive project in itself, our main motivation for this task is the extrapolation for SLAM in higher-fidelity virtual and physical environments.
Minecraft serves as a fully controllable testbed for our supervised and unsupervised testing, as there are procedurally generated terrains with large-scale 3D spaces and dynamic lighting conditions.
Compared to the real world, Minecraft provides perfect map data and ground truth poses, adjustable noise models, and repeatable simulation testing in an infinite and stochastic world.
By training machine learning models to conduct SLAM in Minecraft, we can aim for eventual deployment in higher-fidelity virtual worlds and the physical world.
Models validated in Minecraft can be transferred to photorealistic simulators with more realistic physics engines and sensor modules, with the ultimate goal being SLAM in real-world environments.
